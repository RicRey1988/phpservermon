# PHP Server Monitor 4.3.2-hs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver durable authentication, repaired controls, richer safe diagnostics, customizable identity, and unambiguous server status in v4.3.2-hs.

**Architecture:** Extend existing Symfony session/user preferences and the GD image pipeline rather than adding a second framework. Controllers expose sanitized media URLs as Twig globals; deterministic WebP storage owns all filesystem writes. Diagnostic and status view models remain presentation-ready and safe.

**Tech Stack:** PHP 8.5+, Symfony HttpFoundation/DependencyInjection, Twig, PHPUnit 11, Hope UI, vanilla JavaScript, GD/WebP.

## Global Constraints

- Every product version ends in `-hs`.
- No environment variables, request dumps, credentials, or private VAPID keys appear in diagnostics.
- Uploaded images are decoded and re-encoded; original filenames never become paths.
- Explicit logout remains available and revokes persistent authentication.
- Production database content and private configuration are preserved.
