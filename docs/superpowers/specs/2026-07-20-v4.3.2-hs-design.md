# PHP Server Monitor 4.3.2-hs Design

## Goal

Make authentication durable, repair the VAPID and theme controls, improve diagnostics and status clarity, and let administrators/users personalize the system logo and profile photo without weakening upload security.

## Decisions

- Authentication uses a one-year persistent, rotating remember-me token and a sliding native session. Explicit logout still revokes it. The login screen explains this behavior instead of presenting an optional checkbox that would contradict it.
- The VAPID generator is a distinct submit action with `formnovalidate`; it generates the missing public/private pair before the normal form can require the public key. Plain email contacts are normalized to `mailto:`.
- Password/theme controls use explicit SVG sizing, display, and contrast rules in both light and dark themes.
- Uploaded branding and avatars reuse the existing GD normalization pipeline and are stored as deterministic WebP files outside user-controlled filenames. Logo configuration belongs to General settings; avatar configuration belongs to Profile.
- Branding and avatar paths are stored in the existing configuration and user-preference stores, avoiding a schema change.
- PHP information is grouped into runtime, limits, acceleration, and platform sections and never exposes environment variables, request contents, or credentials.
- Filesystem failures name the effective PHP user, affected directory, reason, and safe ownership/permission remedy. Production grants the `apache` group only the write access required by the updater and logs.
- Server cards receive an unmistakable icon, label, colored status strip, and accessible text for online, offline, warning, and paused states. XHR refresh updates the same visual state immediately.

## Data and error flow

Image uploads pass through size/type validation and GD re-encoding, are written atomically, and retain the previous image when processing fails. Removing an image restores the built-in HS/initial fallback. Controller messages describe upload or VAPID failures without exposing stack traces.

## Verification

Regression tests cover persistent session construction, VAPID form behavior, safe PHP keys/groups, actionable filesystem diagnostics, deterministic image storage, template branding/avatar rendering, and status semantics. The full PHPUnit suite, PHP syntax checks, browser checks in both themes, VPS health checks, and authenticated page navigation run before publishing.
